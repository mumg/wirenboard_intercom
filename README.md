# Intercom Android App

Android-приложение для:

- отображения нескольких RTSP-потоков на главном экране;
- перехода в полноэкранный режим по нажатию на поток;
- SIP-аудио/видеозвонков;
- предпросмотра входящего видео до принятия звонка;
- одновременной регистрации на нескольких SIP-серверах.

## Стек

- Kotlin
- Jetpack Compose
- LibVLC для RTSP
- Linphone SDK для SIP audio/video
- GeckoView для отдельного браузерного движка Mozilla

## Где менять конфигурацию

Единый JSON-файл конфигурации:

- `/Users/maxim/work/home/intercom/app/src/main/assets/app_config.json`

## Что уже реализовано

- плитка из RTSP-стримов на главном экране;
- fullscreen-экран отдельного RTSP-потока;
- инициализация SIP-ядра;
- регистрация нескольких SIP-аккаунтов;
- overlay входящего вызова с полноэкранной видео-поверхностью;
- overlay активного звонка с кнопкой завершения.
- первый этап интеграции GeckoView через отдельный `GeckoViewActivity`.

## Важные замечания

- В текущем окружении не был доступен `gradle`, поэтому фактическую сборку я здесь не прогнал.
- Версию `org.linphone:linphone-sdk-android` при первой сборке может понадобиться скорректировать под ваш Maven-репозиторий и целевую версию SDK.
- GeckoView подключен через официальный репозиторий Mozilla `https://maven.mozilla.org/maven2/`; текущая интеграция использует nightly-канал и может потребовать уточнения версии под вашу сборочную среду.
- Для production обычно стоит вынести SIP-аккаунты и список камер в DataStore/Room или в защищенную удаленную конфигурацию, а не хранить их как seed-данные в коде.
